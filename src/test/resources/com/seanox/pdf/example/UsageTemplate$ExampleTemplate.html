header, located in the markup but will be rendered independently and inserted on each page as overlay
section and placeholder for the footer are optional
the markup generator needs the placeholder #[0x23][header[[...#[0x5D5D5D], if it is to be used
the section <header>...</header> is needed when rendering the PDF to create the overlay, if it is to be used

The file name of the template is derived from the template implementation in this example.
Because this is an inner class, the unique file name results.

Output of outlet
#[outlet[[ #[name]
#[street]
#[zipCode] #[location]
![ADDRESS_TEL]: #[phone]
![ADDRESS_FAX]: #[fax]
![ADDRESS_E_MAIL]: #[email]
![ADDRESS_WEB]: #[websiteUrl] ]]]



Output of articles
#[articles[[

#[title]
![ARTICLE_NUMBER]: #[articleNumber]
![ARTICLE_PRICE]: #[price]

]]]


Additional placeholders
#[0x23][page]
current page, can be used in header and footer

#[0x23][pages]
total number of pages, can be used in header and footer

#[0x23][locale]
locale-setting of the PDF creation,
can be used as attribute in the body tag to use the correct font via CSS

#[0x23][...-exists]
each placeholder automatically has an exists counterpart
e.g. for #[0x23][name] also exists #[0x23][name-exists]
This can be used for CSS to react on the presence or absence of placeholders.
